SERIAL EEPROM INFORMATION


FIXING A CORRUPTED SERIAL EEPROM

This information I found was written before the BACKUP function in EvoX was implemented.

There is a 256 byte Serial  EEPROM on the Xbox which stores:

*XBOX VERSION
*KERNEL VERSION
*RAM SIZE
*XBOX SERIAL
*MAC ADDRESS
*ONLINE KEY
*VIDEO STANDARD
*XBE REGION
*HDD KEY
*CONFOUNDER
*HDD MODEL
*HDD SERIAL
*HDD PASSWORD
*XBOX DVD MODEL

The EEPROM is located on the motherboard between the TSOP and another chip.  The EEPROM is a Rhom BR24C02 device (runs on Vcc = 3.3V), and its I2C address code is 0xA8 (that's already shifted left to compensate for the LSB r/w bit, the "real" device code is 0x54).  typedef struct _EEPROMDATA {
    u8           HMAC_SHA1_Hash[20];                     // 0x00 - 0x13 HMAC_SHA1 Hash
    u8           Confounder[8];                          // 0x14 - 0x1B RC4 Encrypted Confounder ??
    u8           HDDKkey[16];                            // 0x1C - 0x2B RC4 Encrypted HDD key
    u8           XBERegion[4];                           // 0x2C - 0x2F RC4 Encrypted Region code (0x01 North America, 0x02 Japan, 0x04 Europe)
    u8           Checksum2[4];                           // 0x30 - 0x33 Checksum of next 44 bytes
    u8           SerialNumber[12];                       // 0x34 - 0x3F Xbox serial number
    u8           MACAddress[6];                          // 0x40 - 0x45 Ethernet MAC address
    u8           UNKNOWN2[2];                            // 0x46 - 0x47 Unknown Padding ?
 
    u8           OnlineKey[16];                          // 0x48 - 0x57 Online Key ?
 
    u8           VideoStandard[4];                       // 0x58 - 0x5B  0x00800300 = PAL, 0x00400100 = NTSC, 0x00400200=NTSC_J
 
    u8           UNKNOWN3[4];                            // 0x5C - 0x5F  Unknown Padding ?
 
    //Comes configured up to here from factory..  everything after this can be zero'd out...
    //To reset XBOX to Factory settings, Make checksum3 0xFFFFFFFF and zero all data below (0x64-0xFF)
    //Doing this will Reset XBOX and upon startup will get Language & Setup screen...
    u8           Checksum3[4];                           // 0x60 - 0x63  other Checksum of next
 
    u8           TimeZoneBias[4];                        // 0x64 - 0x67 Zone Bias?
    u8           TimeZoneStdName[4];                     // 0x68 - 0x6B Standard timezone
    u8           TimeZoneDltName[4];                     // 0x5C - 0x6F Daylight timezone
    u8           UNKNOWN4[8];                            // 0x70 - 0x77 Unknown Padding ?
    u8           TimeZoneStdDate[4];                     // 0x78 - 0x7B 10-05-00-02 (Month-Day-DayOfWeek-Hour)
    u8           TimeZoneDltDate[4];                     // 0x7C - 0x7F 04-01-00-02 (Month-Day-DayOfWeek-Hour)
    u8           UNKNOWN5[8];                            // 0x80 - 0x87 Unknown Padding ?
    u8           TimeZoneStdBias[4];                     // 0x88 - 0x8B Standard Bias?
    u8           TimeZoneDltBias[4];                     // 0x8C - 0x8F Daylight Bias?
 
    u8           LanguageID[4];                          // 0x90 - 0x93 Language ID
 
    u8           VideoFlags[4];                          // 0x94 - 0x97 Video Settings - 0x96 b0==widescreen 0x96 b4 == letterbox
    u8           AudioFlags[4];                          // 0x98 - 0x9B Audio Settings
 
    u8           ParentalControlGames[4];                // 0x9C - 0x9F 0=MAX rating
    u8           ParentalControlPwd[4];                  // 0xA0 - 0xA3 7=X, 8=Y, B=LTrigger, C=RTrigger
    u8           ParentalControlMovies[4];               // 0xA4 - 0xA7 0=Max rating
 
    u8           XBOXLiveIPAddress[4];                   // 0xA8 - 0xAB XBOX Live IP Address..
    u8           XBOXLiveDNS[4];                         // 0xAC - 0xAF XBOX Live DNS Server..
    u8           XBOXLiveGateWay[4];                     // 0xB0 - 0xB3 XBOX Live Gateway Address..
    u8           XBOXLiveSubNetMask[4];                  // 0xB4 - 0xB7 XBOX Live Subnet Mask..
    u8           OtherSettings[4];                       // 0xA8 - 0xBB Other XBLive settings ?
 
    u8           DVDPlaybackKitZone[4];                  // 0xBC - 0xBF DVD Playback Kit Zone
 
    u8           UNKNOWN6[64];                           // 0xC0 - 0xFF Unknown Codes / Memory timing table?
 } EEPROMDATA



The Region Code

This DWORD is encrypted. It corresponds to the region code in the XBE header:

1 North America
2 Japan
4 Europe/Australia
0x80000000 Manufacturing plant

DVD Region

This DWORD corresponds to the region code for playback of DVD movies:

0x00 None
0x01 Region 1
... ...
0x06 Region 6

FIXING A BROKEN SERIAL EEPROM:

How Can I Fix My Broken EEPROM?
First, try EVERYTHING else before you mess with your EEPROM. You can permanently ruin your xbox if you make a mistake. Now then... The following procedure was used to recover a retail xbox after using the XDK recovery disc on it. so it is possible hat you can get it to work on yours... Using an XDK Recovery disk BLANKS the EEPROM, so On to it:

You need to get a valid eeprom.bin for your xbox. You should have backed up your xbox eeprom already, but I know how some of you are, so there is another option. However, if your hard drive is locked, you must have the hard drive unlock key from your xbox EEPROM. There is NO other way. Additionally, if you have a 1.6 xbox with a locked hard drive, I do not know any method of creating a usable bios with embedded eeprom. You will have to wait.. Sorry! You can try to use the method described below to boot your xbox (the hard drive key portion of your eeprom could still be undamaged, but first chance you get, you need to unlock your hard drive, before flashing the eeprom), but there are no guarantees.  If your HDD was unlocked, then you can breathe a little easier.

Yoshiro made a great debug bios. It is bootable on all versions of xbox and has a valid embedded eeprom... This is the key to making this fix easy... Thanks Yoshihiro!

Make Your Own Working EEPROM
Here is what you need to make a working EEPROM file:

   1. LiveInfo Beta2 from TeamAssembly
   2. bunnie's eeprom, or a blank eeprom as provided with LiveInfo

This part is rather simple. Open your chosen eeprom file with LiveInfo. Make sure that the xbox version is set correctly:
1.0 = 1.0 xbox 1.1 = 1.1-1.6 xbox.
Enter the serial number of your xbox (on the sticker on the bottom of the xbox).
Enter an MAC address for your ethernet connection. This has to be different than other MAC addresses on your network. This is bunnie's MAC address: 00-50-F2-4F-65-52.
Set your video/XBE region.
You don't have to put an Online Key, unless you know it from your backup eeprom.
The DVD Kit Zone can be set to 00.

I am not sure what the confounder does exactly, but I used bunnie's before with no problems: 00000000A0F31300.
And the Unique Harddrive key. This must be the same as your previous eeprom if your hard drive has been locked. Otherwise, you may not be able to use the hard drive anymore.

Now save the the eeprom.bin file from the edits you just made.

Making Your Own Bios With An Embedded EEPROM
You will need the following:

   1. Xecuter2 4981 bios (just use the .06 version for now)
   2. xbtool by nghtshd. (xbtool website)
   3. your eeprom.bin file. (with working hard drive unlock code if your hard drive has been locked)

Extract the xbtool rar archive to a folder on your computer. Open the Xecuter2 4981 bios with xbtool. There are several things you can edit, but we are only concerned with the embedded eeprom. You have to place the eeprom file, named "eeprom.bin" in the "Patch Data" folder. then make sure the "Embed Eeprom" box is checked, then save your bios. (also select what size you need the bios to be for you hack...)

Flashing Your EEPROM
Now you are ready to flash the xbox eeprom:

What you need:

   1. Xbox with a modchip (if you have a softmod, or tsop hack, then you will have to find a way to boot something to flash/install a new bios...)
   2. Yoshihiro debug bios 256k Kernel 2005 (This is on xbins) or your own bios with embedded EEPROM as created above.
   3. a working eeprom... either a backup or your xbox eeprom (as you should have backed everything up) or you can use bunnie's eeprom as described above.
   4. EEpromer (This is on xbins)
   5. If you have no replacement dash on your xbox, you need Slayer's Auto Installer, or other bootable disc with ftp server support. (You can make your own! See My Evox Install Guide on instruction to make your own bootable evox disc.)

In the Yoshihiro debug bios 256k Kernel 2005 pack on xbins, there are two bios files. One is for 1.6 xboxes, the other for 1.0-1.5 xboxes. Use the correct version for your xbox. You need to flash this bios to your modchip (or have it load by other means. If you cannot get the xbox to boot at all, then the modchip is really your only option). and start it up... It should look like a normal bios, except the background is white behind the large X logo. NOTE: THIS IS A SPECIAL TYPE OF BIOS! DO NOT USE IT NORMALLY ON YOUR XBOX AFTER REPAIRING THE EEPROM!

If you have a dash installed on your xbox, it should boot up. If not, then try booting Slayer's or another dash disc. You HAVE to get a launcher to load because you need FTP. Your backup EEPROM file should be named eeprom.bin and transferred to the C:\ directory on your xbox. This is where eepromer will look for the new EEPROM file to flash to your xbox.

Before going on, make sure your xbox hard drive is unlocked. If it is locked, you may not be able to unlock it unless you are restoring the same eeprom that is currently on your xbox.

You can either install a dash to your xbox (if you don't have one installed already) or you can make a bootable xbox disc with eepromer on it. Start up your xbox with the debug bios and run eepromer. You will have to press A to continue, then A again. It will make a backup of the current eeprom, and save it as C:\backupeeprom.bin. To continue, press A again, then it will attempt to flash the xbox eeprom. When it is done, it will give a status message, press A, then press X to shut down your xbox. Your eeprom has now been flashed! now reflash the modchip to a normal hacked bios like Xecuter2 4981, 4983, or EvoxM8plus and you xbox should boot up. If you xbox is still showing signs of an eeprom problem, then you may not have edited the bios correctly beforehand.
